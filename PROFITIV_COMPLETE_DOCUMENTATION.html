1. Executive Summary

Profitiv: Where Money Meets Motion

Profitiv is a revolutionary two-sided engagement and earnings platform that connects Creators & Brands with Earners through verified, rewarding digital interactions.

Platform Vision

Profitiv transforms the creator economy by enabling:

  • Creators & Brands: Launch verified campaigns with predictable costs and real engagement from authenticated users
  • Earners: Generate controlled income through watching videos, completing courses, and brand interactions
  • Platform: Sustainable revenue through subscriptions, access fees, and marketplace transactions

Core Value Propositions

For Creators & Brands

  • Real engagement from verified users, not bots
  • Predictable campaign costs with templates and analytics
  • Revenue sharing model - use earnings to fund next campaigns
  • Comprehensive dashboard with live performance tracking
  • Stripe Connect integration for seamless payouts

For Earners

  • Multiple earning methods: videos, games, courses, brand tasks
  • Transparent subscription plans with clear earning caps
  • TIV token system convertible to USD or tradeable on marketplace
  • KYC-protected withdrawals via Stripe
  • Weekly, monthly, and annual earning limits prevent system abuse

Market Positioning

Profitiv occupies a unique position at the intersection of:

  • Creator Economy: Empowering content creators with monetization tools
  • Gig Economy: Providing flexible earning opportunities
  • Performance Marketing: Verified, measurable engagement
  • FinTech: Modern payment rails and token economics

2. Product Overview

Platform Architecture

Profitiv operates as a two-sided marketplace with three key stakeholders:

1. Creators & Brands (Supply Side)

  • Create video campaigns with target view counts
  • Purchase TIV packs to fund rewards
  • Set campaign parameters (reward per view, audience targeting)
  • Monitor campaign performance in real-time
  • Receive revenue share from access fees

2. Earners (Demand Side)

  • Subscribe to a plan ($25-$100/month)
  • Complete verified tasks to earn TIVs
  • Convert TIVs to USD at platform rate
  • Sell TIVs on marketplace for potentially higher rates
  • Withdraw earnings via Stripe (requires KYC)

3. Profitiv Platform

  • Facilitates all transactions and verifications
  • Manages subscription billing and payouts
  • Operates TIV marketplace with transaction fees
  • Ensures compliance and prevents fraud
  • Provides analytics and reporting

Core Features

Campaign Management

Campaign Creation Flow:

  1. Creator uploads video and sets parameters
  2. Defines target views and reward per view (in TIVs)
  3. Purchases TIV packs to fund the campaign
  4. Sets optional audience targeting criteria
  5. Publishes campaign to active earners

Campaign Tracking:

  • Real-time view count and completion rate
  • Average watch duration analytics
  • Remaining budget and TIV pool
  • Engagement metrics and user feedback

TIV Token Economics

TIV (Time Investment Value) is Profitiv's internal currency:

  • Acquisition: Creators purchase TIV packs ($100-$10,000 packages)
  • Distribution: TIVs awarded to earners for completed tasks
  • Conversion: Platform rate typically 1 TIV = $2 USD
  • Trading: Marketplace allows peer-to-peer TIV trading
  • Withdrawal: Convert to USD and withdraw via Stripe

Subscription System

Both Creators and Earners operate on subscription plans that define:

  • Earning Caps: Maximum earnings per period (weekly/monthly/annual)
  • Campaign Limits: Number of active campaigns (Creators)
  • Feature Access: Premium features and priority support
  • Marketplace Perks: Reduced fees or better rates

KYC & Verification

Multi-tier verification system:

  1. Email Verification: Required for account creation
  2. Phone Verification: SMS OTP for enhanced security
  3. Identity Verification: Photo ID + selfie (required for withdrawals)
  4. Address Verification: Proof of address document (for higher limits)

4. Revenue Share Model

Platform Revenue Streams

1. Subscription Revenue (Primary)

Recurring monthly income from both Earners and Creators:

  • Earner subscriptions: $25-$100/month per user
  • Creator subscriptions: $35-$140/month per creator
  • Target: 10,000 earners + 1,000 creators = $450K-$1.14M MRR

2. Marketplace Fees

Transaction fees on TIV pack purchases and peer-to-peer trading:

  • TIV Pack Sales: 5-10% platform fee on creator purchases
  • P2P Trading: 3-5% fee on marketplace transactions
  • Example: $1M in monthly TIV sales = $50K-$100K platform revenue

3. Access Fees (Campaign Participation)

Small fees for earners to participate in premium campaigns:

  • Optional per-campaign access fees ($1-$5)
  • Portion funds TIV reward pools
  • Revenue split: 30% Creator, 40% Rewards, 30% Platform

Sustainability Model

How Profitiv Sustains Operations

Profitiv is built on a balanced ecosystem where:

  • Brands grow through verified engagement
  • Creators expand their reach and earn revenue share
  • Earners are rewarded for authentic participation
  • Platform maintains sustainability through subscriptions and fees

Key: Revenue comes from recurring subscriptions and transaction fees, NOT from investments or pooled funds. This ensures long-term platform independence and transparency.

5. Profitiv Pulse Design System

Design Philosophy

Theme: Futuristic FinTech × Creator Economy — "Where Money Meets Motion"

Core Design Principles

  • Motion & Energy: Subtle animations convey activity and value generation
  • Glassmorphism: Frosted glass effects create depth and modern aesthetics
  • Neon Accents: Electric purple and cyan create futuristic feel
  • Dark Foundation: Deep carbon base allows neon elements to pop
  • Clarity: Despite visual richness, information hierarchy stays clear

Visual Elements

1. Motion Background

Slowly drifting gradient background (15-25 second loop):

  • Creates sense of constant activity
  • Subtle enough not to distract from content
  • Radial gradients positioned strategically
  • Low opacity to maintain readability

2. Glow Effects

Two-layer glow system for depth:

  • Brighter Inner Glow: More saturated, tighter radius
  • Darker Outer Glow: Less saturated, wider radius
  • Applied to cards, buttons, and interactive elements
  • Intensifies on hover/active states

3. Glass Cards

Signature component style:

  • Semi-transparent background with backdrop blur
  • Border with subtle white/color tint
  • Inner shadow for depth
  • Hover state increases brightness slightly

4. Typography

Font Family: Poppins (or similar geometric sans-serif)

  • Headings: 600-700 weight, tight letter-spacing
  • Body: 400-500 weight, comfortable line-height (1.6-1.8)
  • Numbers/Stats: 700 weight, larger size, gradient fills
  • Micro-copy: 400 weight, reduced opacity (0.7-0.8)

5. Buttons & CTAs

Gradient-filled buttons with hover effects:

  • Primary: Gradient fill (purple → cyan)
  • Secondary: Transparent with colored border
  • Rounded corners (999px for pill shape)
  • Gentle pulse animation on primary CTAs
  • Shadow glow increases on hover

Dual Identity System

Earner Interface

  • Color Emphasis: Brighter cyan accents
  • Mood: Energetic, action-oriented
  • Higher neon contrast for engagement prompts
  • Focus: Earning opportunities, progress tracking

Creator Interface

  • Color Emphasis: More purple tones
  • Mood: Professional, analytical
  • Slightly calmer but still energetic
  • Focus: Campaign performance, ROI metrics

Accessibility

  • Contrast Ratios: All text meets WCAG AA standards (4.5:1 minimum)
  • Motion: Respect prefers-reduced-motion media query
  • Focus States: Clear keyboard navigation indicators
  • Screen Readers: Semantic HTML and ARIA labels

Logo Variants

1. Color Gradient (Primary)

Use on dark backgrounds (#0b0c10 or darker)

  • Full gradient with glow effects
  • Highest visual impact
  • Primary usage for headers, hero sections

2. White on Dark

Simplified version for small sizes or contexts

  • Solid white (#FFFFFF) with subtle glow
  • Better readability at small sizes
  • Use in navigation, footers

3. Dark on Light (When Necessary)

For light backgrounds (avoid if possible)

  • Dark purple (#6C63FF) solid color
  • No glow effects
  • Use sparingly, platform is dark-themed

Clear Space & Sizing

  • Minimum Clear Space: 0.5× logo height on all sides
  • Minimum Size: 120px width for digital use
  • Maximum Size: No limit, scales beautifully
  • Aspect Ratio: Maintain original proportions always

Usage Guidelines

✅ DO

  • Use on dark backgrounds for maximum impact
  • Maintain gradient direction (left to right)
  • Keep glow effects subtle and tasteful
  • Ensure sufficient contrast with background
  • Use as primary brand identifier across all touchpoints

❌ DON'T

  • Don't add additional effects or modifications
  • Don't change gradient colors or direction
  • Don't place on busy backgrounds
  • Don't stretch or distort proportions
  • Don't create icon-only versions

Technology Stack

Core Database Tables

subscription_plans

Defines available subscription tiers and their limits:

CREATE TABLE subscription_plans ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), name TEXT NOT NULL, role TEXT NOT NULL, -- 'earner' or 'creator' price DECIMAL(10,2) NOT NULL, weekly_cap BIGINT, monthly_cap BIGINT, annual_cap BIGINT, max_campaigns INTEGER, max_target_views INTEGER, revenue_share_percent DECIMAL(5,2), features JSONB, payout_delay_days INTEGER, active BOOLEAN DEFAULT true, created_at TIMESTAMPTZ DEFAULT NOW() );

user_subscriptions

Tracks user subscription status and caps:

CREATE TABLE user_subscriptions ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), user_id UUID REFERENCES auth.users(id), plan_id UUID REFERENCES subscription_plans(id), role TEXT NOT NULL, status TEXT DEFAULT 'active', kyc_verified BOOLEAN DEFAULT false, started_at TIMESTAMPTZ DEFAULT NOW(), expires_at TIMESTAMPTZ, created_at TIMESTAMPTZ DEFAULT NOW() ); -- RLS Policy ALTER TABLE user_subscriptions ENABLE ROW LEVEL SECURITY; CREATE POLICY "Users can view own subscription" ON user_subscriptions FOR SELECT USING (auth.uid() = user_id);

Key PostgreSQL Functions

participate_in_campaign

CREATE OR REPLACE FUNCTION participate_in_campaign( _campaign_id UUID ) RETURNS JSON AS $$ DECLARE _user_id UUID; _existing_participation UUID; BEGIN _user_id := auth.uid(); -- Check if already participating SELECT id INTO _existing_participation FROM campaign_participants WHERE campaign_id = _campaign_id AND user_id = _user_id; IF _existing_participation IS NOT NULL THEN RETURN json_build_object( 'success', false, 'message', 'Already participating in this campaign' ); END IF; -- Create participation record INSERT INTO campaign_participants ( campaign_id, user_id, progress, started_at ) VALUES ( _campaign_id, _user_id, 0, NOW() ); RETURN json_build_object( 'success', true, 'message', 'Successfully joined campaign' ); END; $$ LANGUAGE plpgsql SECURITY DEFINER;

complete_campaign_view

CREATE OR REPLACE FUNCTION complete_campaign_view( _campaign_id UUID ) RETURNS JSON AS $$ DECLARE _user_id UUID; _reward_per_view DECIMAL(10,2); _campaign_status TEXT; _remaining_budget DECIMAL(10,2); BEGIN _user_id := auth.uid(); -- Get campaign details SELECT status, remaining_budget, reward_per_view INTO _campaign_status, _remaining_budget, _reward_per_view FROM campaigns WHERE id = _campaign_id; -- Verify campaign is active and has budget IF _campaign_status != 'active' OR _remaining_budget < _reward_per_view THEN RETURN json_build_object( 'success', false, 'message', 'Campaign not available' ); END IF; -- Update participant progress UPDATE campaign_participants SET progress = progress + 1, total_earned_tiv = total_earned_tiv + _reward_per_view, total_earned_usd = total_earned_usd + (_reward_per_view * 2), last_activity_at = NOW() WHERE campaign_id = _campaign_id AND user_id = _user_id; -- Update campaign stats UPDATE campaigns SET current_views = current_views + 1, remaining_budget = remaining_budget - _reward_per_view WHERE id = _campaign_id; -- Log earnings INSERT INTO earnings_history ( user_id, campaign_id, activity_type, amount_tiv, amount_usd ) VALUES ( _user_id, _campaign_id, 'video_view', _reward_per_view, _reward_per_view * 2 ); RETURN json_build_object( 'success', true, 'earned_tiv', _reward_per_view, 'earned_usd', _reward_per_view * 2 ); END; $$ LANGUAGE plpgsql SECURITY DEFINER;

10. Stripe Integration Guide

Overview

Profitiv uses Stripe Connect for creator payouts and standard Stripe for earner withdrawals.

Setup Steps

1. Create Stripe Account

  1. Sign up at stripe.com
  2. Complete account verification
  3. Get API keys from Dashboard → Developers → API keys
  4. Use Test mode keys for development

2. Enable Stripe Connect

  1. Dashboard → Connect → Get Started
  2. Choose "Standard" accounts (recommended)
  3. Configure branding and settings
  4. Set up webhooks endpoint

3. Environment Variables

// Initialize Stripe Connect for creator export async function initializeStripeConnect(userId: string) { try { const account = await stripe.accounts.create({ type: 'standard', metadata: { profitiv_user_id: userId } }); const accountLink = await stripe.accountLinks.create({ account: account.id, refresh_url: `${BASE_URL}/stripe-onboarding`, return_url: `${BASE_URL}/dashboard`, type: 'account_onboarding', }); // Save account ID to database await supabase .from('profiles') .update({ stripe_account_id: account.id }) .eq('user_id', userId); return accountLink.url; } catch (error) { console.error('Stripe Connect error:', error); throw error; } }

TIV Pack Checkout

// Create checkout session for TIV pack purchase export async function createTIVPackCheckout( packId: string, userId: string ) { const { data: pack } = await supabase .from('tiv_packs') .select('*') .eq('id', packId) .single(); if (!pack) throw new Error('Pack not found'); const session = await stripe.checkout.sessions.create({ payment_method_types: ['card'], line_items: [ { price_data: { currency: 'usd', product_data: { name: pack.pack_name, description: `${pack.amount_tiv} TIVs`, }, unit_amount: Math.round(pack.price_usd * 100), }, quantity: 1, }, ], mode: 'payment', success_url: `${BASE_URL}/marketplace?success=true`, cancel_url: `${BASE_URL}/marketplace?canceled=true`, metadata: { user_id: userId, pack_id: packId, amount_tiv: pack.amount_tiv.toString(), }, }); return session.url; }

Process Withdrawal

// Process earner withdrawal via Stripe export async function processWithdrawal( withdrawalId: string ) { const { data: withdrawal } = await supabase .from('withdrawal_requests') .select('*, profiles(stripe_account_id)') .eq('id', withdrawalId) .single(); if (!withdrawal) throw new Error('Withdrawal not found'); if (!withdrawal.kyc_verified) { throw new Error('KYC verification required'); } const transfer = await stripe.transfers.create({ amount: Math.round(withdrawal.net_amount * 100), currency: 'usd', destination: withdrawal.profiles.stripe_account_id, metadata: { withdrawal_id: withdrawalId, user_id: withdrawal.user_id, }, }); // Update withdrawal status await supabase .from('withdrawal_requests') .update({ status: 'completed', stripe_transfer_id: transfer.id, completed_at: new Date().toISOString(), }) .eq('id', withdrawalId); return transfer; }

Webhook Handler

// Handle Stripe webhooks export async function handleStripeWebhook( body: string, signature: string ) { const event = stripe.webhooks.constructEvent( body, signature, process.env.STRIPE_WEBHOOK_SECRET! ); switch (event.type) { case 'checkout.session.completed': const session = event.data.object; await handleSuccessfulCheckout(session); break; case 'transfer.paid': const transfer = event.data.object; await handleTransferCompleted(transfer); break; case 'account.updated': const account = event.data.object; await handleAccountUpdated(account); break; } } async function handleSuccessfulCheckout(session: any) { const { user_id, pack_id, amount_tiv } = session.metadata; // Credit TIVs to user await supabase.rpc('credit_user_tiv', { _user_id: user_id, _amount: parseFloat(amount_tiv), _source: 'pack_purchase', }); // Log transaction await supabase.from('tiv_transactions').insert({ user_id, pack_id, amount_tiv: parseFloat(amount_tiv), amount_usd: session.amount_total / 100, stripe_session_id: session.id, status: 'completed', }); }
